# Changelog

本文件遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/) 格式。

## [2.2.0] — 2026-06-28

### 移除

- **输入源从四种缩减为两种**：删除"项目分析模式"和"组合模式"。这两个模式从定义之初即为占位符（无展开的执行逻辑）。对应 README、examples、templates 中的引用已同步清理。

### 新增

- 排错维度下新增可选提示：用户如果手头有相关的 git diff 或 commit，可以在复盘时手动贴入辅助还原。Skill 不会主动扫描代码仓库，由人筛选。

### 变更

- **安装方式简化**：原先四种安装方式（一句话安装 / VS Code / CLI / 全局安装）合并为"一句话安装"一条。所有平台通用，零门槛。
- **Quick Start 重构**：Section 3 精简为 3 个子节（前置依赖 → 安装 → 基本用法）。Section 5（安装与配置）缩减为轻量配置说明，删掉与 Quick Start 重复的安装步骤和跨平台表格。
- **基本用法示例修正**：示例从"从我的项目里生成面试题"改为 `[对话名称]` 引用格式（如"复盘昨天的 Docker 排错对话"），明确题源是 Agent 对话而非项目代码。
- **画像标签重设计**：`经验等级（初级/中级/高级）` 改为 `准备阶段（基础积累/能力提升/查漏补缺）`，不再按工龄划分，按面试准备进度划分。
- **难度分布调整**：基础积累 35/50/15，能力提升 15/55/30，查漏补缺 10/45/45。核心逻辑：进阶题是所有阶段的主战场，面试不因基础题答得好而放行。
- **平台定位明确**：README badge 从 Claude Code → 通用 AI Agent，简介强调本质是平台无关的 system prompt，前置依赖从"Claude Code v1.0+"改为"任意 AI Agent"。

### 修复

- SKILL.md 底部"附加资源"中的 `templates.md` 和 `examples.md` 链接修复为正确路径（`assets/templates.md` 和 `references/examples.md`）。

### 设计原则

v2.2.0 的主题是**收缩与聚焦**。从 v2.1.0 的"全面铺开"回到一个更锐利的位置——Skill 只做一件事：从 Agent 对话中提炼能力。砍占位模式让边界清晰，砍安装方式让门槛更低，砍工龄标签让用户准确自评。不是功能退步，是把力气用在刀刃上。

## [2.1.0] — 初始发布

### 核心功能

- 三种处理模式：拦截记录（5 维度 × 6 要素知识提取）、面试题生成（10 类题型 + 反事实推理 + 复述强制）、知识蒸馏（6 个持久化文件 + SM-2 间隔复习）
- 9 道防御机制：用户确认、范围限制、复述强制、渐进式提醒、信号过滤、深度标注、去重合并、过期审查、冷启动引导
- 难度自适应：根据用户准备阶段（基础积累/能力提升/查漏补缺）调整题目分布
- 知识衰减管理：90/180/365 天审查触发 + 归档机制
